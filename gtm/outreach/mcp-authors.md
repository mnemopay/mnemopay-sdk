## Methodology + gaps

**Method.** Started from public `awesome-mcp-servers` curated lists, filtered to community-built repos with discoverable GitHub topics or `mcp-server` in the name. Every row verified live via GitHub API on 2026-04-09 — star counts and `pushed_at` timestamps are real API responses, not curated-list blurbs. For each author, pulled `/users/{login}` for contact vectors.

**Exclusions.** Anthropic/official MCP servers, Apollo/Redis/MongoDB/GitHub/Intuit/Apify/Hookdeck/Alchemy/LaunchDarkly/Brave, Harness/SonarSource/ChromeDevTools/Jenkins, and any repo whose MCP surface was a thin reference/demo wrapper.

**Gaps:**
1. **Email coverage ~50%.** For ~20 authors the only reliable contact is the GitHub profile (reply via Issues/Discussions/Sponsor). ~15% expose Twitter.
2. **Company affiliation:** kept a handful (Amazon, Microsoft, NS1) only where the repo is clearly a personal side project.
3. **"Real work" filter is qualitative** — biased toward repos where the tool maps to a cost-bearing action. A few borderline (Quran, Open Library, Met Museum) kept because the *author* looks monetizable.
4. **Not fetched:** per-commit activity. `pushed_at` is the proxy for "active." All entries pushed within last 90 days except hamflx/imagen3-mcp.
5. **Target count:** 54 verified entries.
6. **Hooks are specific for top 15, pattern-matched for the long tail.** Spend 30 seconds on each repo's README before sending to tighten the hook to a named tool.
7. **Second-pass source:** `nborwankar/awesome-mcp-servers-2` claims 2,379 MCP servers as of April 2025 — scrape + re-verify if pushing past 60.
